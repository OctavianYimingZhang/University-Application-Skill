# Data Lifecycle

1. Intake user profile and target route.
2. Collect official sources.
3. Extract structured requirements with source provenance, application cycle, access date, and independent availability, verification, completeness, and staleness fields.
4. Collect applicant values as unconfirmed evidence records; never seed them from examples or programme data.
5. Build programme comparison, task, deadline, and document views only after the relevant evidence gate.
6. Mark missing, partial, unverified, conflicted, wrong-cycle, or stale evidence as a blocker.
7. Refresh requirements before final submission, payment, or visa-readiness decisions.
