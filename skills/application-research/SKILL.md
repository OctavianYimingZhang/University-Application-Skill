# Application Research

Read `references/evidence-contract.md` and `references/research.md`.

## Workflow

1. Use the scope already stated by the applicant and ask only for a missing field that changes the research.
2. Open current official pages for the selected programmes and related admissions information.
3. Verify the exact award, structure, application cycle, requirements, costs, deadlines, and requested fit factors.
4. Keep programme identity, official requirements, applicant evidence, and strategic fit as separate layers.
5. Mark unpublished, unavailable, stale, or conflicting facts as gaps.
6. Return the requested shortlist, comparison, direct explanation, source log, cleaned table, or workbook.

Use `catalogues/index.json` as an identity index. Verify selected records against current official pages before relying on availability or requirements.

When fit matters, compare verified programme or supervisor information with confirmed applicant interests and experience. Treat topic relevance as fit evidence and report place availability as a separately verified fact.

Use the user's requested output language; otherwise use English.
