# Submission Readiness

Use this focused Skill for final pre-submission checks.

## Workflow

1. Read [`../../references/submission.md`](../../references/submission.md), [`../../references/quality-checks.md`](../../references/quality-checks.md), and [`../../references/evidence-contract.md`](../../references/evidence-contract.md).
2. Confirm programme, application cycle, current document inventory, and portal status. Ask for citizenship, destination, or funding only when a submission item is genuinely visa- or funding-sensitive.
3. Check programme selection, portal fields, documents, language tests, references, fees, scholarships, deadlines with their stated timezone, mandatory pre-application steps, and visa-sensitive items.
4. Separate blockers from optional improvements.
5. Mark each item with a normalized evidence record or a gap. Apply the `submission` evidence purpose: a private local document may prove presence, but user confirmation alone cannot satisfy readiness, and current official requirements still require official provenance.
6. Return a final action list without implying submission has occurred.

## Output

Produce a concise readiness summary, blockers, date-sensitive tasks, and source log.

Default to English. Use another output language only when the user explicitly requests it.
